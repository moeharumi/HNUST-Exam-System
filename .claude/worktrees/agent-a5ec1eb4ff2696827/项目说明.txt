--------------------------------------------------------------------------------
2.1  启动入口
--------------------------------------------------------------------------------

【main.py】（启动入口）
  这是程序的"大门"，你运行 python main.py 就是从这里开始的。
  它只有一行代码：调用 app.py 里的 run() 函数。

  怎么改：基本不用改这个文件。

--------------------------------------------------------------------------------
2.2  hnust_exam/app.py（总开关）
--------------------------------------------------------------------------------

  这是程序启动后的"总开关"，它做了这些事：
  - 创建窗口（QApplication）
  - 读取你的设置（深色模式、字体大小等）
  - 设置全局样式（颜色、字体、按钮样式等）
  - 打开主窗口
  - 如果程序崩溃了，会自动记录日志

  怎么改：
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 想改默认窗口大小？  → 搜索 "resize(1200, 800)"，改成你想要的数字      │
  │ 想改字体？          → 搜索 "Microsoft YaHei"，换成其他字体名           │
  │ 想改全局颜色？      → 修改 _generate_stylesheet() 里的颜色代码         │
  │ （颜色代码是十六进制，比如 #ff0000 是红色，去网上搜"颜色选择器"就行）  │
  └─────────────────────────────────────────────────────────────────────────┘

2.3  hnust_exam/utils/（工具箱）
--------------------------------------------------------------------------------

【utils/constants.py】（常量 - 存放固定不变的值）
  存放程序用到的所有"固定值"：
  - CURRENT_VERSION = "v1.0.5"  ← 版本号，发新版本时改这里
  - GITHUB_USERNAME = "RyanTanC"  ← GitHub 用户名
  - GITHUB_REPO_NAME = "HNUST-Exam-System"  ← GitHub 仓库名
  - EXAM_TIME_SECONDS = 3600  ← 考试总时间（秒），60分钟=3600秒
  - QUESTION_TYPE_ORDER  ← 题型的显示顺序

  怎么改：
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 想改考试时间？      → 把 EXAM_TIME_SECONDS 改成你要的秒数              │
  │                        比如 30分钟 = 1800，90分钟 = 5400                │
  │ 想改版本号？        → 把 CURRENT_VERSION 改成新版本号                  │
  │ 想改题型顺序？      → 修改 QUESTION_TYPE_ORDER 列表里的顺序            │
  └─────────────────────────────────────────────────────────────────────────┘

【utils/helpers.py】（工具函数 - 一些通用小功能）
  提供一些通用的小功能：
  - get_resource_path()  ← 找到资源文件的路径（打包后也能找到）
  - normalize_answer()   ← 标准化答案（比如判断题"对"和"A"都算同一个）
  - normalize_code()     ← 标准化代码（去掉多余空格和空行）
  - version_tuple()      ← 把版本号 "v1.0.5" 变成 (1,0,5) 方便比较
  - get_log_dir()        ← 找到日志文件夹的位置

  怎么改：一般不用改。如果发现判断题的答案比对有问题，修改 normalize_answer()
  里的 mapping 字典。

【utils/theme.py】（主题 - 控制颜色和字体）
  控制整个程序的颜色主题：
  - _LIGHT = { ... }  ← 浅色模式的所有颜色
  - _DARK = { ... }    ← 深色模式的所有颜色

  怎么改：
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 想改主色调？        → 修改 _LIGHT 里的 "PRIMARY": "#0078d7"          │
  │                        同时也要改 _DARK 里的 "PRIMARY"                 │
  │ 想改背景色？        → 修改 "BG" 的值                                   │
  │ 想改文字颜色？      → 修改 "TEXT" 的值                                 │
  │ 想改成功/错误颜色？ → 修改 "SUCCESS"（绿色）和 "DANGER"（红色）       │
  │                                                                       │
  │ ⚠️ 注意：_LIGHT 和 _DARK 里都有同样的键，两个都要改！               │
  │ （颜色代码去搜"HTML颜色选择器"，选好颜色复制十六进制值就行）         │
  └─────────────────────────────────────────────────────────────────────────┘

2.4  hnust_exam/models/（数据模型 - 存放数据结构）
--------------------------------------------------------------------------------

【models/question.py】（题目数据）
  定义一道题长什么样：
  - index: 题目序号
  - number: 题号（比如 "1", "T1"）
  - q_type: 题型（单选/填空/判断/程序填空/程序改错/程序设计）
  - text: 题目内容
  - options: 选项（{"A": "选项内容", "B": "..."}）
  - correct_answer: 正确答案
  - score: 分值
  - program_file: 程序文件名

  怎么改：如果要新增题型字段，在这里加，然后在其他地方也对应修改。

【models/result.py】（判分结果）
  定义判分结果长什么样：
  - question_number: 题号
  - is_correct: 是否正确
  - user_answer: 用户的答案
  - correct_answer: 正确答案

【models/exam.py】（考试模型 - 核心数据管理）
  这是整个考试的"大脑"，管理：
  - 从 Excel 加载试卷（load_from_excel）
  - 用户答案的存储（answer_map）
  - 题目标记（marked_indices）
  - 当前题目位置（current_index）
  - 判分（调用 grader.py）

  怎么改：
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 想改Excel的读取方式？    → 修改 load_from_excel() 方法                │
  │ 想改Excel的必要列？      → 修改 utils/constants.py 里的 REQUIRED_COLUMNS │
  │ 想改分组逻辑？          → 修改 load_from_excel() 里的分组部分          │
  └─────────────────────────────────────────────────────────────────────────┘

2.5  hnust_exam/services/（服务层 - 干具体活的）
--------------------------------------------------------------------------------

【services/grader.py】（判分服务 - 最重要！）
  负责判断用户的答案对不对：

  - check_fill_in()          ← 检查填空题（支持多个空，用分号分隔）
  - check_program_answer()   ← 检查程序题（精确比对 + 模糊比对）
  - grade_exam()             ← 对整份试卷判分，返回每题结果

  怎么改：
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 填空题判分有问题？                                                   │
  │   → 修改 check_fill_in()：                                           │
  │     当前逻辑是用 ",;，；" 分割后逐个比较                            │
  │     如果你的填空用其他符号分隔，修改 re.split() 里的正则表达式       │
  │                                                                       │
  │ 程序题判分太严/太松？                                                │
  │   → 修改 check_program_answer()：                                    │
  │     最后的 "return seq.ratio() >= 0.9" 是相似度阈值                  │
  │     0.9 = 90%相似才算对，改成 0.8 就是80%相似就算对                 │
  │     改成 1.0 就是必须完全一样                                        │
  │                                                                       │
  │ 判断题答案比对有问题？                                               │
  │   → 去 utils/helpers.py 修改 normalize_answer() 里的 mapping         │
  └─────────────────────────────────────────────────────────────────────────┘

【services/config_manager.py】（配置管理）
  负责读写设置文件（存在 C:\Users\你的用户名\.hnust_exam\ 目录下）：
  - config.json    ← 你的设置（字体、深色模式、即时反馈开关等）
  - progress.json  ← 试卷进度（哪些做过了、最高分多少）

  怎么改：
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 想加一个新的设置项？                                                 │
  │   1. 在 DEFAULTS 字典里加默认值，比如 "my_setting": False            │
  │   2. 在 load_config() 里它会自动加载                                  │
  │   3. 在设置对话框里加对应的 UI 控件                                   │
  └─────────────────────────────────────────────────────────────────────────┘

【services/python_env.py】（Python环境查找）
  负责找到你电脑上的 Python：
  - find_system_python()  ← 自动搜索所有可能的 Python 安装位置
  - open_with_idle()      ← 用 IDLE 打开 .py 文件

  怎么改：一般不用改。如果发现找不到 Python，可能是安装路径太特殊了，
  可以在 common_paths 列表里添加新的搜索路径。

【services/backup_manager.py】（备份管理）
  程序文件的备份和恢复：
  - 开始考试时，把所有用到的 .py 文件备份到 _backup_programs 目录
  - 点"重做"时，从备份恢复原始文件
  - 交卷后，删除备份目录

  怎么改：一般不用改。

【services/update_checker.py】（更新检查）
  去 GitHub 检查有没有新版本：
  - check_update_async()  ← 在后台线程检查更新
  - fetch_update_info()   ← 调用 GitHub API 获取最新版本信息

  怎么改：
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 想改更新检测的地址？                                                 │
  │   → 修改 utils/constants.py 里的 GITHUB_USERNAME 和 GITHUB_REPO_NAME │
  │                                                                       │
  │ 想关掉更新检测？                                                     │
  │   → 在 app.py 里找到 _check_updates_async() 的调用，注释掉即可      │
  └─────────────────────────────────────────────────────────────────────────┘

第三部分：怎么修改——常见操作指南
================================================================================

--------------------------------------------------------------------------------
3.1  我想改考试时间
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/utils/constants.py
  2. 找到 EXAM_TIME_SECONDS = 60 * 60
  3. 改成你要的时间（单位是秒）：
     · 30分钟 = 1800
     · 60分钟 = 3600
     · 90分钟 = 5400
  4. 保存，重新运行 main.py 即可

--------------------------------------------------------------------------------
3.2  我想改主题颜色
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/utils/theme.py
  2. 在 _LIGHT 字典里找到你想改的颜色，比如：
     "PRIMARY": "#0078d7"  ← 这是主色调（蓝色）
     "DANGER": "#ff6600"   ← 这是错误色（红色）
     "SUCCESS": "#28a745"  ← 这是成功色（绿色）
  3. 改成你想要的颜色（用十六进制颜色码，百度搜"颜色选择器"）
  4. _DARK 字典里同样的键也要改
  5. 保存，重新运行

--------------------------------------------------------------------------------
3.3  我想改欢迎页的介绍文字
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/views/welcome_page.py
  2. 找到 sections = [...] 这个列表
  3. 修改里面的 title 和 content，比如：
     ("软件介绍", [
         "这是第一行介绍",
         "这是第二行介绍",
     ]),
  4. 保存，重新运行

--------------------------------------------------------------------------------
3.4  我想加一个底部按钮
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/views/exam_page.py
  2. 找到 btn_configs = [...] 这个列表
  3. 在里面加一行，格式是：("按钮文字", self.你的方法名)
  4. 然后在 ExamPage 类里写对应的方法，比如：
     def my_new_button(self):
         from PySide6.QtWidgets import QMessageBox
         QMessageBox.information(self, "提示", "你点了新按钮！")

--------------------------------------------------------------------------------
3.5  我觉得程序题判分太严格
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/services/grader.py
  2. 找到 check_program_answer() 函数
  3. 找到最后一行：return seq.ratio() >= 0.9
  4. 把 0.9 改成 0.8（80%相似就算对）或 0.7（70%相似）
  5. 保存，运行测试验证：在终端执行
     .venv/Scripts/python.exe -m pytest tests/test_grader.py -v

--------------------------------------------------------------------------------
3.6  我想改倒计时变红的时间
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/views/exam_page.py
  2. 找到 _update_time_display() 方法
  3. 找到 "if self.remaining_time <= 300:"（300秒=5分钟变红）
     改成 600 就是10分钟变红，改成 120 就是2分钟变红
  4. 找到 "elif self.remaining_time <= 600:"（600秒=10分钟变橙）
     这是警告色的时间，也改成你想要的

--------------------------------------------------------------------------------
3.7  我想改字体大小的范围
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/views/dialogs/settings_dialog.py
  2. 找到 self._scale_slider.setRange(80, 150)
  3. 改成你想要的范围，比如 setRange(60, 200)
  4. 保存

--------------------------------------------------------------------------------
3.8  我想改版本号
--------------------------------------------------------------------------------
  1. 打开 hnust_exam/utils/constants.py
  2. 把 CURRENT_VERSION = "v1.0.5" 改成新版本号
  3. 保存

--------------------------------------------------------------------------------
3.9  我想加一套新的试卷
--------------------------------------------------------------------------------
  1. 把 Excel 文件放到 "题库/" 文件夹里
  2. Excel 必须包含这些列：题号、题型、题目、正确答案、分值
  3. 可选列：选项A、选项B、选项C、选项D（单选题用）、程序文件
  4. 重新运行程序，在选卷页就能看到了

--------------------------------------------------------------------------------
3.10 我改了颜色但是有些地方没变
--------------------------------------------------------------------------------
  颜色有两种方式设置的：
  1. theme.py 里的颜色 → 控制大部分界面
  2. 每个页面里硬编码的 setStyleSheet → 这些要单独改

  解决方法：
  1. 打开出问题的页面文件（比如 welcome_page.py）
  2. 搜索 "setStyleSheet" 或 "color:"
  3. 把里面写死的颜色值也改成你想要的

================================================================================
第五部分：在 PyCharm 里打包成 exe
================================================================================

打包就是把你的 Python 程序变成一个 .exe 文件，让没有安装 Python 的人也能用。

--------------------------------------------------------------------------------
5.1  安装打包工具（PyInstaller）
--------------------------------------------------------------------------------
  1. 打开 PyCharm 底部的 "Terminal"（终端）
  2. 输入以下命令安装 PyInstaller：

     .venv/Scripts/pip.exe install pyinstaller

  3. 等待安装完成（可能需要几分钟）

--------------------------------------------------------------------------------
5.2  准备打包

--------------------------------------------------------------------------------
  打包前确认以下内容：
  ✅ 程序能正常运行（python main.py 没有报错）
  ✅ 题库文件夹里有试卷
  ✅ icon.ico 图标文件在项目根目录

--------------------------------------------------------------------------------
5.3  开始打包（推荐方式）
--------------------------------------------------------------------------------
  在 PyCharm 终端输入以下命令（整行复制粘贴）：

 .venv/Scripts/pyinstaller.exe --onefile --windowed --icon=icon.ico --name "HNUST仿真平台" --add-data "题库;题库" --add-data "icon.ico;." --collect-all PySide6 --hidden-import PySide6.QtCore --hidden-import PySide6.QtGui --hidden-import PySide6.QtWidgets --hidden-import PySide6.QtNetwork --runtime-tmpdir "%TEMP%\HNUST_simulation_temp" main.py

  参数解释：
  --onefile     = 打包成单个 .exe 文件（不加的话会生成一堆文件）
  --windowed    = 运行时不显示黑色命令行窗口
  --icon=icon.ico = 使用 icon.ico 作为程序图标
  --name "HNUST仿真平台" = 生成的 exe 文件名叫这个
  --add-data "题库;题库" = 把题库文件夹一起打包进去
  main.py       = 要打包的入口文件

--------------------------------------------------------------------------------
5.4  找到打包好的 exe
--------------------------------------------------------------------------------
  打包完成后，exe 文件在：
  dist/HNUST仿真平台.exe

  你可以把这个 exe 文件复制到任何地方使用。

--------------------------------------------------------------------------------
5.5  如果打包失败，常见问题和解决方法
--------------------------------------------------------------------------------

  问题1："ModuleNotFoundError: No module named 'xxx'"
  解决：在打包命令末尾加上 --hidden-import=xxx
       比如：--hidden-import=openpyxl

  问题2："题库文件夹找不到"
  解决：确认 --add-data 参数格式正确：
       Windows 用分号：--add-data "题库;题库"
       Mac/Linux 用冒号：--add-data "题库:题库"

  问题3："打包后运行闪退"
  解决：
  1. 先不加 --windowed 打包，这样能看到报错信息
  2. 命令改为：
     .venv/Scripts/pyinstaller.exe --onefile --icon=icon.ico --name "HNUST仿真平台" --add-data "题库;题库" main.py
  3. 运行 exe，看黑色窗口里的报错信息
  4. 根据报错信息修复

  问题4："打包后太大了"
  解决：这是正常的，PyInstaller 会把 Python 解释器一起打包。
       一般 50-100MB 是正常的。

  问题5："想减少打包体积"
  解决：加 --exclude-module 排除不需要的模块：
     .venv/Scripts/pyinstaller.exe --onefile --windowed --icon=icon.ico --name "HNUST仿真平台" --add-data "题库;题库" --exclude-module matplotlib --exclude-module scipy main.py

--------------------------------------------------------------------------------
5.6  更简单的打包方式（用 .spec 文件）
--------------------------------------------------------------------------------
  项目里已经有一个 HNUST仿真平台.spec 文件（旧版的）。
  你可以重新生成一个新的：

  1. 先执行一次上面的打包命令，会自动生成 .spec 文件
  2. 下次打包可以直接用：
     .venv/Scripts/pyinstaller.exe HNUST仿真平台.spec
  3. 如果改了代码，只需要重新打包 main.py 和 hnust_exam 文件夹就行

--------------------------------------------------------------------------------
5.7  打包后分发给别人
--------------------------------------------------------------------------------
  把以下文件一起发给别人：
  1. dist/HNUST仿真平台.exe  ← 主程序
  2. 题库/文件夹             ← 如果没打包进去的话

  如果用了 --onefile，就只需要发一个 .exe 文件。
  题库已经在里面了，别人双击就能用。
